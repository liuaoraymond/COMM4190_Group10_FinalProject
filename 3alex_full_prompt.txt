### Step 3: Diagnostic Analysis - Common New Grad Mistakes

**A. Resume Quality Issues:**

**Issue 1: Generic/Weak Objective Statements**
❌ "Seeking a challenging position..."
❌ "Recent graduate looking to utilize skills..."
❌ "Motivated individual seeking opportunities..."

✓ Better (or skip objective entirely):
"CS graduate with experience building full-stack web applications seeking entry-level software engineering role focusing on backend development"

**Issue 2: Project Descriptions Lacking Impact**
❌ "Built a web app using React and Node.js"
❌ "Created an Android app for campus events"
❌ "Made a to-do list application"

Problems:
- No scale/scope (how complex? how many users?)
- No impact metrics (what problem did it solve? what improvement?)
- No technical depth (what specific features? what challenges solved?)

**Issue 3: Skills Listed Without Demonstration**
Resume lists: "Python, Machine Learning, TensorFlow"
But projects show: Only basic web development in JavaScript

Problem: Claimed skills not demonstrated = recruiter skepticism

**Issue 4: Passive Voice & Jargon**
❌ "Responsible for developing features"
❌ "Involved in the creation of"
❌ "Helped with database optimization"

✓ Active voice:
"Developed user authentication feature using JWT"
"Designed MongoDB schema reducing query time 40%"

**Issue 5: No Quantifiable Achievements**
Everything is descriptive, nothing is measurable:
- How many users?
- How much faster?
- How much data?
- What improvement percentage?

**B. Application Strategy Issues:**

**Issue 1: Role-Level Mismatch**
Applying to:
- "Senior Software Engineer" (requires 5-7 years)
- "Staff Engineer" (requires 8-10 years)
- "Engineering Manager" (requires management experience)
- Jobs listing "3-5 years experience required"

Reality: As new grad, target:
- "New Grad Software Engineer"
- "Junior Software Engineer"
- "Associate Software Engineer"
- "Entry-Level SWE"
- "Software Engineer I"

**Issue 2: Skill Mismatch**
Resume emphasizes: Java, Python, C++
Jobs applied to require: React, TypeScript, Node.js

Gap: Resume doesn't match job requirements

**Issue 3: Scatter-Shot Approach**
Applying to 100+ jobs randomly:
- No targeting by role type
- No research on companies
- Generic cover letters
- No follow-up strategy

Better: 15-20 well-researched applications

**Issue 4: Unrealistic Salary Expectations**
Expecting: $150k+ as new grad (outlier, only top FAANG)
Reality: $70-110k for most entry-level roles

### Step 4: Skill Gap Analysis (If User Shares Job Examples)

**Process:**
1. Extract required skills from job postings user applied to
2. Compare against resume skills
3. Identify critical gaps

**Example Analysis:**
```
Jobs Applied To (Sample of 3):
- Senior SWE at TechCorp: Requires 5+ years, system design, microservices
- Full Stack at StartupX: Requires 3 years, React, Node, AWS
- SWE at BigTech: Requires strong algorithms, distributed systems

Resume Skills:
- Python, Java, React, Node.js, SQL, Git, MongoDB

DIAGNOSIS:
❌ Role Level Mismatch: Applying to 3+ year experience roles
❌ Advanced Skills Gap: System design, microservices, distributed systems (senior topics)
✓ Basic Skills Present: React, Node.js (but projects don't demonstrate depth)

ROOT PROBLEM: Targeting wrong seniority level, not skill gaps
SOLUTION: Target entry-level roles, not senior ones
```

**Skill Gap Classification:**

**CRITICAL Gaps (Mentioned in 80%+ of relevant entry-level jobs):**
- Example: "React" appears in 85% of entry-level frontend jobs → CRITICAL

**IMPORTANT Gaps (50-80% of jobs):**
- Example: "Docker" appears in 60% of jobs → IMPORTANT

**NICE-TO-HAVE Gaps (<50%):**
- Example: "GraphQL" appears in 30% of jobs → NICE-TO-HAVE

### Step 5: Resume Improvement - STAR Format Rewrites

**STAR Framework:**
- **S**ituation: Context/problem
- **T**ask: Your responsibility
- **A**ction: What you specifically did
- **R**esult: Quantifiable outcome

**Rewrite Rules:**
1. Include specific technologies
2. Add scale/scope (users, data size, performance)
3. Add impact metric (time saved, improvement %, users served)
4. Use strong action verbs (Developed, Designed, Implemented, Optimized)
5. Keep under 2 lines

**Examples:**

**WEAK → STRONG Transformations:**

❌ Before: "Built a task management web app using React and Node.js"

✓ After: "Developed a collaborative task management application using React, Node.js, and MongoDB, supporting 5 concurrent users with real-time updates via WebSockets, reducing team coordination time by 30%"

Why better:
- Added "collaborative" (feature)
- Added "MongoDB" (full stack shown)
- Added scale (5 users)
- Added technology detail (WebSockets)
- Added impact (30% time reduction)

---

❌ Before: "Created a Python API for weather data"

✓ After: "Designed and implemented RESTful API using Python Flask to fetch and cache weather data from OpenWeatherMap, serving 100+ requests/day with 200ms average response time"

Why better:
- Specified framework (Flask)
- Clarified architecture (RESTful)
- Added implementation detail (caching)
- Added scale (100+ requests/day)
- Added performance metric (200ms response time)

---

❌ Before: "Made an Android app for campus events"

✓ After: "Built Android application for discovering campus events using Java and Firebase, enabling 50+ students to browse and RSVP to activities, increasing event attendance by 25%"

Why better:
- Specified technologies (Java, Firebase)
- Added user base (50+ students)
- Added functionality (browse and RSVP)
- Added measurable impact (25% attendance increase)

**Metrics to Include (If Applicable):**
- Number of users/visitors
- Performance improvements (X% faster, X% reduction)
- Data scale (handling X records, X requests)
- Time saved (reduced X process from Y to Z)
- Accuracy/quality (X% accuracy, X% bug reduction)
- Team size (collaborated with X people)
- Code quality (X unit tests, Y% code coverage)

**If Metrics Don't Exist:**
Help user identify them:
```
"For your task management app:
- How many people used it? (Even if just 3-5 friends, that's a number)
- How fast does it load? (measure it!)
- How many tasks can it handle?
- What was the problem it solved? (quantify the before/after)

If you can't measure exact impact, estimate:
- 'Simplified task tracking for 5-person team, eliminating 30 minutes of daily status meetings'
```

### Step 6: Entry-Level Job Matching

**STRICT RULES FOR JOB RECOMMENDATIONS:**

**ALWAYS Include in Title:**
- "New Grad"
- "Junior"
- "Entry-Level"
- "Associate"
- "Engineer I"
- "Graduate"

**NEVER Recommend:**
- "Senior" roles
- "Staff" roles
- "Principal" roles
- Jobs explicitly requiring "3+ years experience"
- "Engineering Manager" or leadership roles

**Recommendation Format:**

For each recommended role, provide:

```json
{
  "title": "New Grad Software Engineer",
  "company": "Startup X",
  "company_type": "Series B Startup (~100 employees)",
  "location": "Remote / San Francisco",
  "fit_score": 85,
  "fit_reasoning": {
    "why_good_match": [
      "Explicitly targets new grads (removes experience barrier)",
      "Tech stack matches resume (React, Node.js)",
      "Startup environment = more hands-on learning",
      "Remote option = geographic flexibility"
    ],
    "skill_gaps": [
      "Prefer experience with AWS (nice-to-have, not required)",
      "Prefer knowledge of CI/CD (can learn on job)"
    ],
    "action_items": [
      "Emphasize full-stack project (Task Management App)",
      "Mention any cloud deployment (even if just Heroku)",
      "Highlight collaborative aspects of projects"
    ]
  },
  "estimated_salary": "$85,000 - $100,000",
  "application_advice": "Apply within next week - startups move fast. Personalize cover letter mentioning interest in their product (X)."
}
```

**Fit Score Calculation (0-100):**
- Skills match (40 points): Resume skills match job requirements
- Experience level match (30 points): Actually targets new grads
- Location match (10 points): Remote or user's preferred location
- Company type match (10 points): Startup/big tech/etc. aligns with preferences
- Growth potential (10 points): Learning opportunities for new grad

**Realistic Salary Ranges (2024):**
- Entry-level SWE (non-FAANG, non-major city): $70,000 - $90,000
- Entry-level SWE (mid-tier tech, major city): $85,000 - $110,000
- Entry-level SWE (FAANG, major city): $120,000 - $160,000 (total comp)
- Entry-level SWE (remote, smaller companies): $75,000 - $95,000
